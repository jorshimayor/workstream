# Pilot Plan

## Pilot Goal

Prove that Workstream can run real tasks through the complete lifecycle and expose operational failures early.

## Pilot Scope

Use 3 project templates:

1. Technical Task Creation
2. Technical Review
3. Data QA / Evaluation QA

Avoid broad categories until the workflow is stable.

## Pilot Participants

- 1 admin
- 2 operators
- 2 reviewers
- 1 payment/reconciliation owner

## Pilot Task Volume

Minimum:

- 20 tasks created
- 10 submissions
- 5 review decisions
- 3 needs-revision loops
- 2 paid records

## Pilot Measurements

Track:

- time from ready to submitted
- time from submitted to review decision
- checker failure rate
- needs-revision rate
- acceptance rate
- reviewer disagreement
- payout pending amount
- operator confusion points

## Adversarial Drills

Seed negative or edge-case packets during the pilot:

- fake or irrelevant evidence
- missing revision replay row
- critical- or high-severity checker failure with admin override request
- reviewer disagreement on review decision
- imported task rejected during screening
- payment amount mismatch
- accepted task missing evidence citation

## Pilot Rules

- no task without project guide
- no submission without evidence
- no review without checker results
- no accepted task without contribution record
- no payable accepted contribution without its CompensationAward
- no manual status change without audit note

## Pilot Report

The final report includes:

- what worked
- what failed
- missing checkers
- unclear review rules
- payment tracking issues
- dashboard mismatches
- v0.2 priorities
